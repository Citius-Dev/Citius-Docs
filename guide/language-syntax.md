# 语法参考

## 注释

Citius 支持四种注释风格，覆盖了不同场景的需求：

```c
// 行注释（C++ 风格）—— 最常用，适用于单行说明
# 行注释（Shell 风格）—— 适用于快速注释单行代码
/* 块注释（C 风格） */    —— 适用于行内或短段注释

<!--- 命名块注释（Citius 特有） --->
// 上面这种是 Citius 独有的命名块注释
// <!--- 和 ---> 之间的内容会被编译器忽略
// 适合暂时屏蔽大段代码
```

## 类型系统

### 基本类型

Citius 的类型系统直接映射到 C99 的 `stdint.h` 标准类型，每种类型在编译后都变成对应的 C 类型，不产生额外运行时开销。

| Citius 类型 | C 类型 | 说明 |
|-------------|--------|------|
| `void` | `void` | 无类型，用于函数无返回值 |
| `bool` | `bool` | 布尔值，值为 `true` / `false` |
| `i8` | `int8_t` | 有符号 8 位整数，范围 \[-128, 127\] |
| `i16` | `int16_t` | 有符号 16 位整数，范围 \[-32768, 32767\] |
| `i32` | `int32_t` | 有符号 32 位整数，范围 \[-2³¹, 2³¹-1\] |
| `i64` | `int64_t` | 有符号 64 位整数，范围 \[-2⁶³, 2⁶³-1\]，默认整数类型 |
| `u8` | `uint8_t` | 无符号 8 位整数 |
| `u16` | `uint16_t` | 无符号 16 位整数 |
| `u32` | `uint32_t` | 无符号 32 位整数 |
| `u64` | `uint64_t` | 无符号 64 位整数 |
| `f32` | `float` | 32 位单精度浮点数 |
| `f64` | `double` | 64 位双精度浮点数，默认浮点类型 |

**设计意图：** 类型名直接反映位宽（如 `i32` 就是 32 位整数），比 C 的 `int`/`long`/`short` 更清晰、跨平台行为一致。

### 复合类型

- **指针类型：** `Type*` 表示指向 Type 的指针；`Type* mut` 表示指针本身可变（默认不可变）
- **数组类型：** `Type[...]` 表示元素类型为 Type 的数组
- **字符串：** 内置别名 `str`，编译后对应 C 的 `char*`

### 字面量

| 字面量 | 示例 | 编译器推断类型 |
|--------|------|----------------|
| 整数 | `42`, `0`, `12345` | `i64`（编译为 `int64_t`） |
| 浮点数 | `3.14`, `1e-5` | `f64`（编译为 `double`） |
| 字符串 | `"hello\n"` | `str`（编译为 `char*`） |
| 字符 | `'x'`, `'\n'` | `i8`（编译为 `int8_t`） |
| 布尔 | `true`, `false` | `bool`（编译为 `_Bool`） |

字符串和字符中支持的转义序列：`\n`（换行）、`\t`（制表符）、`\\`（反斜杠）、`\"`（双引号）、`\'`（单引号）、`\0`（空字符）。

**推理规则：** 当你写 `x = 42` 而不指定类型时，编译器会自动将 `x` 推断为 `i64`，因为整数字面量的默认类型是 `i64`。

## 变量

Citius 的变量设计遵循**默认不可变**原则，鼓励编写更安全、更可预测的代码。

```c
// var 关键字显式声明变量，带类型注解 i32，值为 10
// 没有 mut 关键字，所以 x 不可变（类似 C 的 const）
var x: i32 = 10

// mut 关键字使变量可变，可以重新赋值
// y 是类型 i32 的可变变量
mut y: i32 = 20
y = 30          // 可以修改，因为 y 是 mut 声明的

// 不带 var 的赋值会"自动声明"变量（Python 风格）
// z 之前没有出现过，此语句会声明 z 并赋值为 42
// 类型自动推断为 i64（整数字面量默认类型）
z = 42

// 自动声明的变量默认是"可变"的
name = "Citius"   // 推断为 str（char*）
name = "NewName"  // 可以重新赋值
```

**规则总结：**
- `var x: Type = value` —— 显式声明，**不可变**（相当于 C 的 `const Type x = value`）
- `mut x: Type = value` —— 显式声明，**可变**
- `x = value`（x 首次出现）—— 自动声明并赋值，**可变**，类型推断
- `pub var x: i32 = 10` —— `pub` 关键字使变量变为**模块外部可见**

## 函数

函数是 Citius 的基本组织单元。参数类型和返回类型均可选，极大简化了函数定义。

```c
// 基本函数：接收两个参数 a 和 b，b 标注为 i32 类型
// 返回类型标注为 i32
// 花括号内是函数体
func add(a, b: i32) -> i32 {
    return a + b    // return 语句返回值
}

// 参数类型可选：name 没有类型注解
// 编译器会尝试从调用处推断 name 的类型
// 返回类型也省略，默认推断为 i64
func greet(name) {
    // 字符串拼接调用 print
    print("Hello, " + name)
}

// 递归函数：参数 n 类型省略，默认推断为 i64
// 返回类型也省略（默认 i64）
// 函数体内部调用自身
func fib(n) {
    if n <= 1 { return n }           // 递归基
    return fib(n - 1) + fib(n - 2)   // 递归调用
}

// pub 关键字使函数对外可见
// 默认所有函数是 static（文件内可见）
// main 通常需要 pub 才能被外部链接
pub func main() -> void {
    print("Hello!")
}
```

**函数设计要点：**

| 特性 | 说明 |
|------|------|
| 关键字 | `func`（`fn` 是已弃用的别名） |
| 参数 | 用逗号分隔，类型注解可选（`name: Type`） |
| 返回类型 | 箭头语法 `-> Type`，可选，默认 `i64` |
| 可见性 | 默认 `static`（文件内），`pub` 全局可见 |
| 入口 | `main` 函数是程序入口，推荐返回 `void` |

## 控制流程

### if / else —— 条件分支

条件表达式不需要圆括号包裹，花括号是必需的：

```c
// condition 是布尔表达式，不加 ()
if condition {
    // condition 为 true 时执行
} else if condition {
    // 前一条件不满足且当前条件为 true 时执行
} else {
    // 所有条件都不满足时执行
}
```

### while —— 条件循环

```c
i = 0               // 初始化循环变量
// 每次迭代前检查条件，条件为 false 时退出
while i < 10 {
    printi(i)       // 输出当前 i 的值
    i = i + 1       // 更新循环变量
}
```

**执行流程：** 检查条件 → 为 true 则执行代码块 → 回到检查条件 → 为 false 则退出循环。

### for（范围循环）

`for x in range` 是 Python 风格的迭代，编译后生成等效的 C `for` 循环：

```c
// 从 0 迭代到 9（10 次），i 依次取值 0,1,2,...,9
// 语法：for 变量 in 起始值..结束值（不包含结束值）
for i in 0..10 {
    printi(i)       // 输出 0 到 9
}

// 快捷写法：省略起始值，默认为 0
// 等价于 for i in 0..5
for i in 5 {
    printi(i)       // 输出 0 到 4
}
```

**编译后的 C 代码等价于：** `for (int64_t i = 0; i < 10; i++) { ... }`。循环变量固定为 `i64`（`int64_t` 类型）。

### break / continue —— 循环控制

```c
i = 0
while true {        // 无限循环
    i = i + 1
    if i == 3 {
        continue     // 跳过本次循环剩余部分，进入下一次迭代
    }
    if i > 5 {
        break        // 立即退出循环
    }
}
// 执行到这里时 i = 6（i=3 被跳过，i=6 触发 break）
```

**区别：** `continue` 跳过当前迭代进入下一次；`break` 完全终止循环。

## 结构体

结构体将多个字段组合为一个复合类型，映射到 C 的 `struct`。

```c
// 定义一个名为 Point 的结构体
// 包含两个 f64 字段 x 和 y
struct Point {
    x: f64          // 默认私有（文件内可见）
    y: f64
    pub z: f64      // pub 标记的字段对外可见
}

// pub 标记结构体本身，使其在文件外部也可使用
pub struct User {
    name: str       // 字符串字段
    age: i32        // 年龄字段
}
```

**可见性层级：**
- 结构体字段默认 `private`（仅在当前文件可见）
- `pub` 字段可在其他文件中访问：`point.z`
- `pub struct` 使结构体本身对外可见，其他人可以声明该类型的变量

## 运算符

### 二元运算符（按优先级从高到低）

```c
a * b    // 乘法
a / b    // 除法（整数除法截断）
a % b    // 取模（仅整数类型）

a + b    // 加法
a - b    // 减法

a << b   // 左移 (a * 2^b)
a >> b   // 右移 (a / 2^b)

a < b    // 小于
a > b    // 大于
a <= b   // 小于等于
a >= b   // 大于等于

a == b   // 等于
a != b   // 不等于

a && b   // 逻辑与（短路求值：a 为 false 则不计算 b）
a || b   // 逻辑或（短路求值：a 为 true 则不计算 b）

a = b    // 赋值（右结合：a = b = c 等价于 a = (b = c)）
```

### 一元运算符

```c
-expr    // 取负：改变数值符号
!expr    // 逻辑非：true 变 false，false 变 true
~expr    // 按位取反：对每一位取反（0 变 1，1 变 0）
```

### 后缀表达式

```c
func_name(args)    // 函数调用：执行函数并传递参数
object.member      // 成员访问：访问结构体的字段
array[index]       // 数组索引：访问数组的第 index 个元素（从 0 开始）
```

## 修饰符与声明

### pub —— 公开可见性控制

`pub` 修饰符控制标识符的可见性，默认所有声明都是文件内可见（C 的 `static`）。

```c
pub func main() -> void {}   // 函数全局可见
pub struct Config {}          // 结构体全局可见
pub x: i32 = 10              // 变量全局可见
```

### import / in —— 模块导入

```c
import "module_name"   // 导入其他模块
in "module_name"       // 'in' 是 'import' 的别名，两者完全等价
```

> ⚠️ **当前状态：** `import` 语句在词法和语法分析阶段被正确解析并在 AST 中注册，但代码生成器尚未生成对应的 C `#include` 指令。这是一个待实现特性。

### otherhave —— 外部语言互操作声明（规划中）

```c
// 声明该模块与其他语言的互操作关系
// 目前仅做解析和记录，不产生代码
otherhave(C, Python)
```

`otherhave` 是 Citius 规划的**语言互操作**特性的标记语句，用于声明当前模块需要与哪些外部语言交互。目前已被解析器识别并存储到 AST 中，但代码生成阶段尚未实现对应功能。

### annotate —— 元数据注解

```c
// 为当前模块附加元数据，供外部工具使用
// 键值对格式，值必须是字符串
annotate {
    symbol1: "value1"
    type: "br"
}
```

`annotate` 用于给模块附加上下文元数据，类似于 C 的 `__attribute__` 或 Java 的注解。目前被解析器识别并存储到 AST 中，但代码生成器尚未处理。

### package —— 包声明

```c
// 声明当前文件所属的包名
// 用于模块组织和命名空间管理
package("myapp")
```

## 语句分隔

Citius 允许使用**分号**或**换行符**作为语句分隔符，减少打字量，提升代码可读性：

```c
a = 1; b = 2     // 一行放多条语句时需要用分号分隔
c = 3             // 换行自动作为语句分隔，不需要写分号
d = 4             // 连续换行表示连续的多条语句
```

**编译器行为：** 词法分析器在遇到换行时会发出一个特殊 token，解析器将其视为语句结束标记。这意味着：
- 一条语句占用一行时不需要写分号（类似 Go、Python）
- 一行内放多条语句时需要分号分隔
- 语句末尾加分号也是允许的（兼容 C 习惯）

## 类型推断与提升

Citius 有一个简单但实用的类型推断系统，减少显式类型注解的负担：

```c
// 1. 字面量推断
x = 42          // x 推断为 i64（整数字面量默认 i64）
y = 3.14        // y 推断为 f64（浮点字面量默认 f64）
name = "hello"  // name 推断为 str（char*）

// 2. 混合类型运算时的类型提升
z = 42 + 3.14   // 42（i64）被提升为 f64，z 推断为 f64
                // 等价于 C 的 double z = (double)42 + 3.14

// 3. 函数参数类型推断
func add(a, b) { return a + b }
// 当调用 add(1, 2) 时，a 和 b 被推断为 i64
// 当调用 add(1.0, 2.0) 时，a 和 b 被推断为 f64
```

**提升规则：**
| 场景 | 规则 |
|------|------|
| 整数 + 整数 | 结果类型为 `i64` |
| 浮点 + 浮点 | 结果类型为 `f64` |
| 整数 + 浮点 | 整数提升为 `f64`，结果类型为 `f64` |
| 函数参数 | 从调用处的实参类型推断，需所有调用点一致 |
