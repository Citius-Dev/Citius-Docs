# Citius 语言简介

## 什么是 Citius？

**Citius**（拉丁语，意为"更快"）是一门**编译到 C** 的高层次系统级编程语言。它的名字源自奥林匹克格言 *Citius, Altius, Fortius*（更快、更高、更强），体现了对性能的不懈追求。

**核心理念：用 Python 的语法写代码，得到 C 级别的运行性能。**

Citius 不是解释型语言，也不是运行在虚拟机上的语言。它是一个**源到源编译器**（transpiler）：你写 `.ct` 文件，Citius 编译器将它转译为标准的 C99 代码，然后由你系统上的 Clang/GCC/MSVC 编译成原生机器码。

### 一句话概括

> Citius = Python 风格的语法 + C 级别的性能 + 语言内置的网络和延时功能

## 设计目标

### 1. 简洁易用

Citius 的语法受 Python 启发，追求"少打字、多做事"：

```c
// Python 风格：变量自动声明，不需要写类型
x = 42
name = "Citius"

// 语句不需要分号，换行就是分隔符
// 条件/循环不需要括号
if x > 0 {
    print("positive")
}
```

相比 C 语言，同样的逻辑 Citius 代码量减少约 30%，且更易阅读。

### 2. 原生性能

Citius 不引入任何运行时开销 —— 所有内建功能（print、sleep、sent）都直接生成对应的 C 代码：

```c
// Citius 源码
print("Hello")
sleep 1 s

// 编译后生成的 C 代码（无额外抽象层，直接调用系统 API）
printf("%s\n", "Hello");
#ifdef _WIN32
    Sleep(1000);
#else
    nanosleep(...);
#endif
```

### 3. 零成本抽象

语言内置的功能（输出、延时、网络请求）在编译后直接变成 C 函数调用，不需要运行时库、垃圾回收器或虚拟机。程序体积小、启动快、内存占用低。

### 4. 渐进式类型

类型注解完全**可选** —— 你可以像 Python 一样完全不写类型，也可以逐步加上类型注解让代码更健壮：

```c
// 阶段 1：快速原型 —— 不写任何类型
func add(a, b) { return a + b }

// 阶段 2：逐步加固 —— 给关键参数加类型
func add(a, b: i32) -> i32 { return a + b }

// 阶段 3：完全类型化 —— 所有参数和返回值都标注
func add(a: i32, b: i32) -> i32 { return a + b }
```

这种方式允许你在快速开发和代码安全性之间灵活切换。

### 5. 默认安全

Citius 的变量默认**不可变**（immutable），使用 `mut` 关键字显式声明可变性：

```c
x = 42          // 不可变：x = 43 会编译错误
mut y = 42      // 可变：y = 43 允许
```

这一设计吸收了 Rust 的理念——默认不可变让代码更可预测、更容易推理，减少意外的状态修改。

## 语言特性一览

| 特性 | 说明 | 设计启示 |
|------|------|----------|
| **语法风格** | 缩进表示块结构，分号可选，换行分隔语句 | Python |
| **类型系统** | 静态类型 + 类型推断，`i8`-`i64`、`u8`-`u64`、`f32`、`f64`、`bool` | C stdint.h |
| **可变性控制** | 默认不可变，`mut` 关键字可变 | Rust |
| **控制流程** | `if/else`、`while`、`for x in range` | Python + C |
| **函数** | `func` 关键字，参数/返回值类型可选，`pub` 可见性 | Go |
| **结构体** | `struct` + 字段类型注解，支持 `pub` 成员 | C |
| **内置输出** | `print`（通用+换行）/ `printi`（整数不换行） | BASIC |
| **延时控制** | `sleep`/`wait` + 人类可读时间单位（ms/s/min/h/d/m/y） | 自定义 |
| **网络请求** | `sent` 语句，语言层面 HTTP/TCP/UDP | 自定义（独创） |
| **编译目标** | C99 代码 → Clang/GCC/MSVC → 可执行文件 | 经典 transpiler |

## 编译流程：从 .ct 到可执行文件

```
foo.ct (Citius 源码，你手写的代码)
  │
  ▼
┌─────────────────────────────┐
│ 1. Lexer (词法分析器)       │
│    将源码拆分为 Token 流     │
│    "func" → 关键字 Token    │
│    "main" → 标识符 Token    │
│    "("   → 分隔符 Token     │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│ 2. Parser (语法分析器)      │
│    递归下降解析              │
│    Token → 抽象语法树 (AST) │
│    检查语法是否正确          │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│ 3. Sema (语义分析器)        │
│    类型检查                 │
│    符号解析（变量、函数）    │
│    类型推断（自动补全类型）  │
│    注册内建函数 (print等)   │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│ 4. CodeGen (代码生成器)     │
│    AST → C99 源码           │
│    插入 #include            │
│    生成辅助函数             │
└─────────────┬───────────────┘
              ▼
      foo.c (C 源码，自动生成)
              │
              ▼
┌─────────────────────────────┐
│ 5. 系统 C 编译器            │
│    Clang / GCC / MSVC       │
│    foo.c → foo.exe          │
└─────────────┬───────────────┘
              ▼
      foo.exe (可执行文件)
```

这个流程每个阶段都是独立的，你可以在任意阶段检查编译器的输出。

## 适用场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **系统编程** | 需要底层控制但不想用 C 的繁琐语法 | 文件操作、进程管理 |
| **网络工具** | 利用 `sent` 内置网络能力快速开发 | HTTP 客户端、网络探测 |
| **教学案例** | 经典四阶段编译器架构，适合学习编译原理 | 课程作业、个人项目 |
| **嵌入式脚本** | 通过 C 中介编译到嵌入式目标平台 | IoT 设备、单片机 |
| **快速原型** | Python 风格语法快速实现想法，后期加固类型 | 算法验证、工具脚本 |
